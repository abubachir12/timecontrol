# 📖 ГАЙД: Деплой TimeControl Bot на сервер

## Что у тебя есть
| Файл | Где работает | Назначение |
|---|---|---|
| `main.py` | Сервер (Ubuntu) | Telegram-бот |
| `server.py` | Сервер (Ubuntu) | API для агента |
| `client.py` | Твой Windows ПК | Агент-трекер |

---

## ЧАСТЬ 1 — Настройка сервера (Ubuntu 22.04)

### Шаг 1. Подключись к серверу по SSH

На Windows открой PowerShell или cmd:
```
ssh root@176.119.158.69
```
Введи пароль от сервера.

---

### Шаг 2. Создай папку проекта и загрузи файлы

```bash
mkdir -p /root/timecontrol
cd /root/timecontrol
```

**Загрузи файлы с компьютера** (выполни в PowerShell на Windows, НЕ на сервере):
```
scp main.py server.py requirements_server.txt root@176.119.158.69:/root/timecontrol/
```

---

### Шаг 3. Установи Python и зависимости

```bash
apt update && apt install -y python3 python3-pip python3-venv

# Создаём виртуальное окружение
cd /root/timecontrol
python3 -m venv venv

# Устанавливаем пакеты
venv/bin/pip install --upgrade pip
venv/bin/pip install -r requirements_server.txt
```

---

### Шаг 4. Вставь токен бота в main.py

Получи токен у @BotFather в Telegram, потом:
```bash
nano /root/timecontrol/main.py
```
Найди строку:
```python
API_TOKEN = "ВСТАВЬ_ТОКЕН_БОТА"
```
Замени на свой токен. Сохрани: `Ctrl+O → Enter → Ctrl+X`

---

### Шаг 5. Открой порт 8000 в файрволе

```bash
ufw allow 8000
ufw allow 22    # чтобы не потерять SSH!
ufw enable
ufw status
```

---

### Шаг 6. Настрой автозапуск через systemd

**Загрузи .service файлы:**
```
scp timecontrol-bot.service timecontrol-server.service root@176.119.158.69:/etc/systemd/system/
```

**Или создай их вручную на сервере:**
```bash
nano /etc/systemd/system/timecontrol-server.service
```
Вставь содержимое из `timecontrol-server.service`, сохрани.

```bash
nano /etc/systemd/system/timecontrol-bot.service
```
Вставь содержимое из `timecontrol-bot.service`, сохрани.

**Активируй сервисы:**
```bash
systemctl daemon-reload

# Запускаем сначала API сервер
systemctl enable timecontrol-server
systemctl start timecontrol-server

# Потом бота
systemctl enable timecontrol-bot
systemctl start timecontrol-bot
```

**Проверь, что всё работает:**
```bash
systemctl status timecontrol-server
systemctl status timecontrol-bot
```
Должно быть `active (running)` зелёным цветом.

---

### Шаг 7. Проверь API сервер

Открой в браузере: `http://176.119.158.69:8000/api/health`
Должен вернуть: `{"status":"ok","time":"..."}` ✅

## ЧАСТЬ 2 — Получить file_id для агента (.exe)

Когда соберёшь .exe из client.py (см. ниже):

1. Открой бота в Telegram → напиши `/start`
2. Перешли `.exe` файл боту как документ
3. Бот ответит: `✅ Файл получен! Вставь в AGENT_FILE_ID:`
4. Скопируй ID
5. Вставь в `main.py`:
   ```python
   AGENT_FILE_ID = "BQACAgIA..."
   ```
6. Перезапусти бота:
   ```bash
   systemctl restart timecontrol-bot
   ```


## ЧАСТЬ 6 — Что делать если что-то не работает

### Бот не отвечает
```bash
systemctl status timecontrol-bot
journalctl -u timecontrol-bot -n 50
```
Скорее всего: неверный токен или не установлены пакеты.

### Статистика не приходит
1. Проверь, запущен ли агент на ПК
2. Проверь доступность API: `http://176.119.158.69:8000/api/health`
3. Проверь порт: `ufw status` — порт 8000 должен быть в списке

### Скриншот не приходит
Агент делает скриншот → отправляет на сервер → бот забирает и шлёт в Telegram.
Задержка до 10 секунд — это нормально.

### После reboot всё пропало
Проверь:
```bash
systemctl is-enabled timecontrol-bot    # должно быть "enabled"
systemctl is-enabled timecontrol-server # должно быть "enabled"
```
Если "disabled" — выполни `systemctl enable timecontrol-bot`.
